# QA Checklist

## 必过项（缺一不合格）

- [ ] 背景是**纯白** `#FFFFFF`，无任何颜色叠加、渐变、纹理、噪点
- [ ] 只有**一种**强调色贯穿全图，无第二色（红色通知点例外，且最多 1 个）
- [ ] 线条**均匀粗细**、干净、无手绘抖动
- [ ] 无任何阴影、发光、模糊、3D、等距轴测感
- [ ] **UI 卡越出后景**，明显遮挡了后景的一部分
- [ ] UI 卡里的文字**具体、可读、和产品相关**（不是 lorem ipsum）
- [ ] UI 卡有窗口顶栏（三点或极简收起按钮）+ 品牌小 logo
- [ ] 至少 2-3 条头像行 / 内容条，头像是**线稿肖像 + 温暖肤色**
- [ ] 有 **AI 提示 pill**（sparkle + 行动文字 + 光标），且在 UI 卡上/边
- [ ] 装饰物 3-5 个，**均匀分布**在 UI 卡周围，不堆一角
- [ ] 主体占画面 55-70%，四周有干净留白
- [ ] 人物只有**皮肤色块 + 黑色头发 + 极简五官**，无腮红、无高光、无夸张表情

## 失败信号（触发即重生成或局部修图）

| 症状 | 处理 |
|---|---|
| UI 卡看起来像通用聊天框，认不出是啥产品 | 用"UI 卡太抽象"编辑提示补充具体内容 |
| 出现了紫+橙、蓝+绿等多色 | 用"强调色跑歪"编辑提示统一 |
| UI 卡下方出现明显阴影 | 用"出现了阴影"编辑提示去掉 |
| 装饰物挤成一团 / 全在一侧 | 用"装饰太多"编辑提示重新分布 |
| 后景把整个画面填满，UI 卡没有"探出来"感 | 强调 "background scene must only occupy 60-70%" 重生成 |
| 人物腮红/闪光眼/大眼睫毛/卡通表情包脸 | 追加 "no blush, no eye highlights, no cartoon expressions — minimal 2-3 stroke faces" |
| 出现了纸纹 / 复古 / 水彩感 | 追加 "clean flat vector only, no texture, no watercolor, no vintage feel" |
| 出现了 isometric / 3D 透视 | 追加 "strict flat 2D, no isometric, no perspective, no depth" |
| AI pill 不见了 / 变成了普通按钮 | 用"AI pill 缺失"编辑提示补上 |
| 图像里出现了 "AI generated" 水印或乱码文字 | 追加 "no watermark, no meta text, only the legible UI content specified" 重生成 |

## 交付判断

高质量图应该让人**一眼觉得：**

1. 这是一个真产品的官网插画（不是概念图，不是设计稿）
2. 我知道这个产品在做什么（UI 卡内容传达了核心场景）
3. 视觉干净、专业、现代（可以放进任何 SaaS 网站不违和）
4. 品牌色的存在感克制但清晰（不刺眼、不喧宾夺主）

如果第一眼像**通用 stock illustration**（那种任何产品都能用的空洞小人图），或者像**设计稿线框**（灰底 + 占位符），就不合格——本 skill 的核心竞争力是"UI 具体 + 场景真实 + 品牌色克制"。
