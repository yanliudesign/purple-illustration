# 构图模式与装饰法

## 5 种主构图（选一个）

每种构图都有**两个层**：后景场景（线稿）+ 前景 UI 卡（仿真产品截图）。UI 卡必须**越出后景边界**（例如挡住笔电的一半、伸出手机边框），形成"漂浮"感。

### A. 笔电 + 前景 UI 卡（默认，最泛用）

- 后景：一台线稿笔电，屏幕里显示 3-6 人的视频会议网格（或产品的次要 UI）
- 笔电旁：一株小盆栽（左下）、极细水平线（桌面）
- 前景：一张竖长方形 UI 卡（约画面高度的 80%），从右侧或右上侵入，遮住笔电右半
- 适合：会议、协作、通讯、录屏、笔记、CRM

### B. 手机 + 前景 UI 卡

- 后景：一部竖着的线稿手机
- 前景：另一张 UI 卡（可以是同 app 的另一屏、或是 web 端配套 UI）从侧边伸出
- 装饰：耳机、咖啡杯、消息通知
- 适合：mobile-first、社交、fintech、健康

### C. 单人 + 前景 UI 卡

- 后景：一个坐姿或站姿的线稿角色（半身），面朝 UI 卡，做"看/指/说"的动作
- 前景：UI 卡漂在角色斜前方
- 装饰：角色头顶漂一个品牌色思考气泡、桌上放一支铅笔
- 适合：创作者工具、写作、设计、单人生产力

### D. 双人协作 + 前景 UI 卡

- 后景：两个线稿人物一坐一站，共同看向前景 UI 卡
- 前景：UI 卡展示两人正在协作的产物（文档、看板、白板）
- 装饰：两个不同方向的对话气泡
- 适合：团队协作、看板、白板、共享文档

### E. 抽象工作空间 + 前景 UI 卡（无人物）

- 后景：一张桌面俯视 / 一个开着的笔电 / 一堆漂浮的功能模块
- 前景：UI 卡为主
- 装饰：小图标群（齿轮、铅笔、放大镜、日历），品牌色气泡若干
- 适合：基础设施、API、开发者工具、数据

## UI 卡的固定要素

无论选哪种构图，UI 卡都要包含以下细节，才够"真"：

- **窗口顶栏**：红/黄/绿三个圆点（macOS 风格）或极简收起/关闭按钮
- **品牌 logo**：左上角一个小圆或图标（品牌色）
- **头像 + 名字 + 时间戳**：至少 2-3 条消息 / 记录 / 条目
- **真实文字内容**：不要 lorem ipsum，用**用户提供的真实产品文案**
- **一个"AI 提示 pill"漂在卡上**：白底、圆角、细品牌色边框、左边一个 sparkle ✦ 图标（品牌色）、右边一个手形光标 🖱️，中间是行动提示文字（如 "Create the next meeting agenda"、"Summarize this thread"）——这是**这类插画最有辨识度的元素**，强烈建议保留

## 散落装饰物库

装饰物**总量 3-5 个**，均匀分布在 UI 卡周围形成"环绕漂浮"，**不要落地**、不要堆一起。

| 装饰 | 画法 | 位置建议 |
|---|---|---|
| 品牌色实心聊天气泡 | 圆角矩形 + 小三角尾巴，内部 2-3 条白色横线代表文字 | UI 卡左上、右上 |
| 空心聊天气泡 | 只有黑色描边，内部 2-3 条黑色横线 | UI 卡右侧 |
| 小铅笔 | 品牌色实心，倾斜 45° | UI 卡下方或左下 |
| Sparkle / ✦ 火花 | 品牌色实心四角星 | UI 卡右上角 |
| 圆圈包裹的第三方 logo | 白底圆 + 黑色细描边 + 中心 logo（如 Teams、Zoom、Slack、Notion 图标） | UI 卡左侧、贴笔电 |
| 小盆栽 | 只有线稿、简单叶子 | 后景笔电左下 |
| 便签 / 卡片碎片 | 品牌色浅色块 + 几条横线 | UI 卡背后半遮 |
| 光标 🖱️ | 黑色手形指针 | 附在 AI pill 右侧 |
| 通知红点 | 只在需要暗示"未读"时用，**唯一允许红色**的场合 | UI 卡内头像角 |

## 层叠规则（这个风格的灵魂）

- **UI 卡永远在最前面**，遮住后景的一部分
- **后景场景永远只画到 60-70%**，剩下 30-40% 被 UI 卡遮住，制造"探出来"的感觉
- **装饰物在 UI 卡和后景之间**穿插（有的在 UI 卡后，有的在 UI 卡前），让画面有厚度
- **地面线**永远在最后景，长度不超过画面宽度的 70%，两头淡出（或直接是一段短线）

## 反流水线规则

不要照搬案例。每次要根据产品特性重新决定：

- UI 卡展示**这个产品最核心的一屏**（不要千篇一律画一个 chat）
- 装饰物**呼应产品品类**（写作产品放铅笔和便签；数据产品放图表碎片；会议产品放摄像头图标和视频窗口）
- 后景**呼应用户使用场景**（远程办公→笔电+咖啡，移动场景→手机+耳机，团队场景→白板+两个人）

## 常见误用

- ❌ UI 卡里画一个通用聊天窗 → 观众不知道你在卖什么
- ❌ 装饰物全部堆在一侧 → 视觉失衡
- ❌ 后景画太满（房间、家具、窗户）→ 变成场景插画，不是产品插画
- ❌ 人物成为视觉焦点 → 应该让产品是焦点，人物是氛围
- ❌ 加了阴影想让 UI 卡"浮起来" → 用层叠和留白，不用阴影
